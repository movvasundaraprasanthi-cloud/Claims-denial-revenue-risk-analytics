# Claims Denial & Revenue Risk Analytics

Industry-standard healthcare revenue cycle analytics project using Snowflake, SQL, and Power BI

## Business Problem
Revenue cycle teams need visibility into denial drivers, rework load, payer performance, and recovery opportunities. Leadership requires a dashboarding layer that shows where denials are concentrated, which categories create the most rework, and how much revenue is at risk.

## Objective
Create an analytics solution that:
- tracks claim denial volume and denial reason trends
- quantifies rework and paid-after-rework outcomes
- highlights payer, region, and specialty-level risk
- supports management decisions through KPI reporting and dashboards

## Tech Stack
- Snowflake
- SQL
- Power BI
- CSV
- GitHub

## Resume-Ready Bullets
- Built a healthcare revenue cycle analytics project to monitor denial trends, rework load, payer performance, and revenue-at-risk indicators
- Developed SQL-based KPI logic to analyze denial reasons, paid-after-rework outcomes, and specialty-level exception patterns
- Created Power BI dashboards to improve visibility into denial drivers, claim recovery trends, and operational bottlenecks
